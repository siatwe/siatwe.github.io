doctrine persisting objects

20170510

back

If the entity class 'Product' is depicted on the corresponding product table, you can save the 'Product' objects to the database. The following example shows how to do this within a controller. In order to check the functionality, the corresponding route has to be called up in the browser.

Example

// ...
use AppBundle\Entity\Product;
use Symfony\Component\HttpFoundation\Response;

// ...
public function createAction()
{
    $product = new Product();
    $product->setName('Keyboard');
    $product->setPrice(19.99);
    $product->setDescription('Ergonomic and stylish!');

    $em = $this->getDoctrine()->getManager();

    // teilt Doctrine mit, dass man eventuell das Product speichern moechte
    $em->persist($product);

    // Fuehrt die Statements aus und speichert (z. B. the INSERT query)
    $em->flush();

    return new Response('Saved new product with id '.$product->getId());
}
