git major commands

back

config

Setzt die Konfigurationswerte fest. Festgelegt werden kann zum Beispiel die Email-Adresse oder der Nutzername.

Example

$ git config --global user.name "John Smith"

init

Initialisiert ein Verzeichnis als git Verzeichnis. Ein leeres Verzeichnis kann mit diesem Befehl als git Verzeichnis identifiziert werden. Dazu wird ein versteckter Ordner (.git) und die Datei .gitignore erstellt.

Example

$ git init

clone

Mit diesem Befehl klont man ein git Verzeichnis von einem Server in das angegebene Verzeichnis auf dem lokalen Computer.

Example

$ git clone https://github.com/siatwe/doc.git

add

Fuegt Dateien dem git Verzeichnis hinzu (Dateien die spaeter commitet werden). Mit der 'git add .' werden alle Dateien die sich gerade im Verzeichnis befinden der Versionskontrolle hinzugefuegt.

Example

$ git add .

rm

Analog wie der add Befehl nur werden in diesem Fall Dateien geloescht.

Example

$ git rm file.txt

commit

Nimmt alle Aenderungen, neue Dateien oder Loeschungen fuegt sie zu einem Objekt zusammen, die dann versehen mit einem Kommentar der Versionskontrolle uebergeben werden.

Example

$ git commit -m "initial commit"

status

Ueberprueft den Zustand der Dateien im git Verzeichnis. Angezeigt wird bspw. ob geaenderte oder neue Dateien noch nicht mit dem commit Befehl der Versionskontrolle uebergeben wurden.

Example

$ git status

branch

Listet existierende Zweige des Verzeichnisses auf.

Example

$ git branch

checkout

Mit diesem Befehl checkt man aus einem anderen Zweig (Branch) aus.

Example

$ git checkout

merge

Fuegt Aenderungen zusammen und erstellt daraus einen neunen Commit.

Example

$ git merge

reset

Setzt das Verzeichnis auf den Stand des letzten Commits zurueck.

Example

$ git reset

fetch

Wenn man seine lokalen Aenderungen wieder auf den Stand des bspw. Master Branches auf dem Server zuruecksetzen moechte, wuerde der Befehl 'git fetch master' lauten.

Example

$ git fetch master

pull

Kombination von fetch und merge. Nimmt die Aenderungen des Servers und fuehrt sie mit den eigenen Aenderungen zusammen.

Example

$ git pull

push

Laedt alle lokalen Aenderungen auf den Server.

Example

$ git push
