Unit Tests Beispiel

Angenommen im Pfad src/AppBundle/Util befindet sich die einfache Klasse Calculator, sollte sich der Unit Test dafuer im Pfad tests/AppBundle/Util befinden. Die Namensgebung ist folgende: KlassennameTest. Die Testklasse erbt die Klasse TestCase, welche mit PHPUnit mitgeliefert wird.

Example Hauptklasse

// src/AppBundle/Util/Calculator.php
namespace AppBundle\Util;

class Calculator
{
    public function add($a, $b)
    {
        return $a + $b;
    }
}

Example Testklasse

// tests/AppBundle/Util/CalculatorTest.php
namespace Tests\AppBundle\Util;

use AppBundle\Util\Calculator;
use PHPUnit\Framework\TestCase;

class CalculatorTest extends TestCase
{
    public function testAdd()
    {
        $calc = new Calculator();
        $result = $calc->add(30, 12);

        // assert that your calculator added the numbers correctly!
        $this->assertEquals(42, $result);
    }
}

Ausf체hren der/des Tests

Alle Tests innerhalb der Applikation ausfuehren (dazu befindet man sich im Root des Verzeichnisses)

Alle Tests im Util Verzeichnis durchfuehren

Alle Tests fuer die Klasse Calculator durchfuehren

Alle Tests im AppBundle durchfuehren

phpunit tests/AppBundle

Functional Tests

Functional Tests testen die Integration der verschiedenen Schichten der Anwendung (vom Routing bis zur View), sie unterscheiden sich nicht von Unit Tests haben aber einen sehr spezifischen Workflow:

  • Mache eine Anfrage
  • Teste die Antwort
  • Klicke auf einen Link oder schicke ein Formular ab
  • Teste die Antwort
  • Zuruecksetzen und wiederholen

Functional Tests werden ueblicherweise in tests/AppBundle/Controller gespeichert und sind einfach PHP Dateien.
