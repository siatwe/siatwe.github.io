git major commands

created: 20161212

updated: 20170619

back

config

Sets the configuration values. The email address or the username can be defined.

Example

$ git config --global user.name "John Smith"

init

Initializes a directory as a git directory. An empty directory can be used with this command as git directory. This creates a hidden folder (.git) and the .gitignore file.

Example

$ git init

clone

Using this command, you clone a git directory from a server to the specified directory on the local directory Computer.

Example

$ git clone https://github.com/siatwe/doc.git

add

Add files to the git directory (files to be committed later). With the 'git add .'- command all files that are currently in the directory will be added to the version control.

Example

$ git add .

rm

Just like the add command, only files are deleted in this case.

Example

$ git rm file.txt

commit

Takes all changes, adds new files or deletions to an object that is then tagged with a comment of the version control.

Example

$ git commit -m "initial commit"

status

Checks the state of the files in the git directory. It shows whether changed or new files still not with the commit command of the version control.

Example

$ git status

branch

List all branches

Example

$ git branch

checkout

Checkout to a branch

Example

$ git checkout

merge

Compile changes and create a new commit.

Example

$ git merge

reset

Resets the directory to the state of the last commit.

Example

$ git reset

fetch

If you reset your local changes to the state of the master branch on the server, the command would be 'git fetch master'.

Example

$ git fetch master

pull

Combination of fetch and merge. Takes the changes of the server and guides them with their own changes together.

Example

$ git pull

push

Launches all local changes to the server.

Example

$ git push
